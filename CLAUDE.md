# WVCAC Website Project

## What this is
A full static website built for **Washita Valley Community Action Council** (washitavalleycac.com), scraped and rebuilt locally with improvements. 15 pages total, ADA/WCAG 2.1 AA compliant.

## Live site
**https://zukkaus.github.io/website-scraper/**

## How pages are generated
- `index.html` — home page, edited directly
- All 14 inner pages are generated by running:
  ```bash
  python3 build_pages.py
  ```
  After any change to `build_pages.py`, run that command and commit.

## Local development server
```bash
cd /Users/miguelvasquez/Documents/website-scraper
python3 -m http.server 8080
```
Then open: http://localhost:8080/index.html

## Deploy to live site
```bash
git add -u
git commit -m "describe change"
git push
```
GitHub Pages auto-deploys within ~1 minute.

## Tech stack
- Pure HTML/CSS/JS — no framework, no build tools
- Python 3 + BeautifulSoup for scraping (`scraper.py`, `build_pages.py`)
- GitHub Pages for hosting (repo: zukkaus/website-scraper)

## Google Sheets integrations (pending setup)
Three pages are wired up to pull live data from Google Sheets CSV.
To connect each one: publish the sheet as CSV, paste the URL in `build_pages.py`, run the generator.

| Page | Variable in build_pages.py | Sheet columns |
|------|---------------------------|---------------|
| Staff | `STAFF_CSV_URL` | Name, Title, Email, Phone |
| Policy Council calendar | `SHEET_CSV_URL` | Date, Time, Title, Description |
| Board agenda | `AGENDA_CSV_URL` | Meeting, Date, Time, Location, Item#, Topic, Notes |

## Azure SSO (Agency Forms — pending setup)
Agency Forms page uses MSAL.js. Needs one-time Azure AD app registration.
Variables to fill in `build_pages.py`:
- `AZURE_CLIENT_ID` — Application (client) ID
- `AZURE_TENANT_ID` — Directory (tenant) ID
Setup instructions are on the agency-forms.html page itself.

## Accessibility
- Custom widget in `accessibility.js` — 11 options matching original site
- Ctrl+U keyboard shortcut opens the menu
- Floating button bottom-right, dark red (#9b1c1c)
- All fonts ≥ 1rem, contrast ratios pass WCAG AA
- 44×44px minimum touch targets throughout

## Key decisions made
- Logo bar at top (95px tall) above sticky navy navbar, gold bottom border
- Menu items centered, font-size .95rem
- Hero: hands/unity image at 2400px (original 4000×2158px) with dark gradient overlay
- Gallery: editorial layout — image + text pairs alternating left/right
- Staff page: 2-column plain text layout matching original (gray background)
- Footer: dark (#111827), explicit colors (no opacity), Facebook icon links to search page
- Annual report 2020-2021: Word doc converted to PDF locally
